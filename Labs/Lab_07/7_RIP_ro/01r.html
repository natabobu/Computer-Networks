Rutarea ca Management al Retelei

O dezvoltare recenta a retelelor de calculatoare si indespensabilitatea lor in viata noastra, devine mai importanta pentru a servi reteaua; in special pentru a mentine reteaua si pentru a rula. De exemplu, administratorul de retea, a carui ocupatie este de a tine reteaua disponibila, trebuie sa fie capabil sa raspunda diferitelor greseli oricand. Deci, este important sa realizam controlul dinamic al retelei pentru a controla disponibiltatea retelei. In acest studiu, noi am analizat unul dintre cele mai populare protocoale de rutare dinamica de cost minim, RIP (Routing Information Protocol) si algoritmul sau.

Protocolul Dinamic de Rutare

Protocoalele de rutare dinamica se impart in urmatoarele categorii:
  • Rutare Unicast
    • IGP (Interior Gateway Protocol)
      • RIP (Routing Information Protocol)
      • OSPF (Open Shortest Path First)
    • EGP (Exterior Gateway Protocol)
      • BGP4 (Border Gateway Protocol version 4)
  • Rutare Multicast
    • DVMRP (Distance Vector Multicast Routing Protocol)
    • MOSPF (Multicast OSPF)
    • PIM (Protocol Independent Multicast protocol)
    • MBGP (Multicast BGP)
Depinzand de topologia de retea, de dimensiunea suprafetei de comunicare, de modul de calculare a cailor cele mai scurte si asa mai departe, algoritmii de rutare dinamica se impart in patru categorii principale:
    1. Distance Vector Algorithm (DV)
    2. Link State (LS)
    3. Path Vector (PV)
    4. Source Routing
