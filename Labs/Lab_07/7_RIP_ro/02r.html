Istoric

RIP (Routing Information Protocol) a fost unul dintre primele intra-AS Internet protocale de rutare si este si azi unul dintre cele mai raspandite. Acesta isi pastreaza originile si numele arhitecturii Xerox Network Systems (XNS). Raspandirea larga a RIP a fost datorata in mare parte companiei Berkeley Software Distribution (BSD), care in 1982 scoate o versiune UNIX care suporta TCP/IP. RIP veriunea 1 este definita in RFC 1058, cu o retro-compatibila versiune 2 definita in RFC 2453.

Distance Vector Routing Algorithm

Sa ne amintim algoritmul Distance Vector (DV), si sa privim caracterisiticile sale.
Algoritmul DV este iterativ, asincron si distribuit unic.
  • El este distribuit in asa fel incat fiecare nod primeste informatii de la unul sau mai multe noduri vecine atasate direct, executa un calcul si apoi distribuie rezultatelele calculului sau in spate la vecinii lui.
  • El este iterativ, deoarece procesul contiuna pana cand nici o informmatie nu mai este schimbata intre vecini.
  • Acest algoritm este asincron deoarece nu necesita toate nodurile pentru a opera in "lockstep" cu fiecare.

Principalele Trasaturi ale RIP

RIP este Vector Distanta protocol care opereaza intr-o maniera foarte apropiata de protocolul idealizat, care permite ruterelor vecine sa schimbe intre ele informatia de rutare. Caracteristicile RIP sunt impartite dupa cum urmeaza:
  • RIP pentru IPv4 ; RFC 1058/1721/1722/1723/1724
  • RIP pentru IPv6 ; RFC 2080
  • Rutare in BSD, SunOS
  • Maximum hopuri; 15
  • Cold-Start ; Ii ia aproximativ 450s(=15x30 ) pentru a calcula.
  • Schimba informatia la fiecare 30 sec. in baza de date Distance Vector (tabelul de rutare) folosind UDP port 520.
    ->transfer conmditionat de schimbarea nodului/linkului
  • Daca nu se raspunde in 180 sec. cade reteaua (keep-alive)
Expresia pentru distanta minima;
D(i,j) ; vector distanta
d(i,j) ; distanta dintre nod_i si nod_j
D(i,j) = min [d(i,k) + D(k,j)] (for all k)

Algoritmul Bellman-Ford (Matematic)

Algoritmul Bellman-Ford a fost dezvolatat de Xerox-PARC ca XNS-RIP. Acesta defineste calea optima catre destinatie: distanta minima si informatia vector distanta de atribut. Acest algoritm este matematic exprimat in tabelul urmator.
